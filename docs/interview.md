# AI 应用工程师面试讲法

AI 应用工程师项目不能只讲“我调用了大模型”。更好的讲法是围绕业务问题、系统架构、核心流程和工程化处理展开。

以知识库问答助手为例，可以先讲普通大模型缺少私有知识、容易幻觉，所以需要 RAG。然后讲系统如何读取文档、切分文本、检索相关片段、把上下文注入 prompt、调用模型生成答案并展示引用来源。

面试官常追问的问题包括：为什么用 RAG、如何减少幻觉、如何优化延迟、如何评估回答质量、如何保护 API key、如何处理模型超时和失败。

前端开发者转 AI 应用工程师时，可以重点讲清楚前后端边界：前端负责输入、状态、结果和引用展示；后端负责鉴权、模型调用、检索、日志、错误处理和成本控制。
