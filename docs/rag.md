# RAG 基础

RAG 是 Retrieval-Augmented Generation 的缩写，中文常叫检索增强生成。它的核心思想是：先从外部知识库检索和问题相关的资料片段，再把这些片段作为上下文交给大模型生成答案。

普通 Chatbot 只依赖模型参数和当前输入，遇到私有资料、项目笔记、实时信息时容易回答不准。RAG 可以把“知识来源”从模型内部迁移到可维护的文档库里，让答案更可追溯。

一个最小 RAG 流程通常包括：文档导入、文本切分 chunk、检索 TopK 片段、把片段拼接到 prompt、调用 LLM 生成答案、返回引用来源。

引用来源最好由后端检索模块返回，而不是让模型自己编造。前端可以单独展示来源文件和命中的片段，方便用户确认答案依据。
